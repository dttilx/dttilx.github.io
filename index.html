01 / AI Network Diagnosis

Current Focus

AI 弱网智能检测

面向延迟、丢包、抖动、连接失败等弱网场景，把传统网络指标和智能分析流程结合起来，输出更容易复盘的检测结果。

状态 本地项目，案例整理中

公开仓库整理后补充源码入口。

背景

弱网问题通常不是单一指标异常，而是延迟、丢包、抖动、链路状态共同作用。项目重点是把检测结果整理成可判断、可追踪、可复盘的工程输出。

实现功能

  • 采集并整理网络质量指标
  • 识别弱网、延迟、丢包和抖动异常
  • 输出检测分级和问题归因辅助
  • 生成检测报告，便于后续复盘
  • 为规则判断和 AI 分析预留扩展接口

技术栈

  • Python
  • AI 判断逻辑
  • Network Diagnostics
  • 异常分析
  • 数据清洗
  • 自动化脚本
  • 报告生成
  • 规则引擎
02 / Distributed System

Systems Project

raftKvstorage

基于 Raft 思想实现的分布式 KV 存储项目，覆盖节点协作、一致性、日志复制、持久化、客户端请求和底层存储结构。

GitHub 查看源码

C++ / Raft / gRPC

背景

分布式 KV 存储需要处理 Leader 选举、日志同步、节点故障、客户端一致性读写等问题。这个项目用于系统化练习共识算法与后端工程组织。

实现功能

  • Raft Leader 选举与任期管理
  • 日志复制、提交与状态机应用
  • KVServer 与客户端 Clerk
  • Persister 持久化组件
  • gRPC / protobuf 通信接口
  • SkipList 底层存储结构

技术栈

  • KV Storage

Workflow

我怎么使用 AI Coding

不是只让 AI 写代码，而是把它放进需求拆解、方案验证、编码、调试和文档整理的完整流程。

01拆需求

把想法拆成模块、输入输出、边界条件和可验收结果。

02搭结构

先定目录、数据流和核心接口，再进入代码实现。

03快迭代

用 Cursor / Codex 辅助实现、重构、补测试和修复问题。

04做交付

整理 README、截图、运行方式和项目复盘，让作品能被别人看懂。
