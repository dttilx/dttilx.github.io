01 / Distributed KV Storage

Core Systems Project

raftKvstorage

基于 Raft 思想实现的分布式 KV 存储项目，覆盖节点协作、一致性、日志复制、持久化、客户端请求和底层存储结构。

GitHub 查看源码

C++ / Raft / gRPC

背景

分布式 KV 存储需要处理 Leader 选举、日志同步、节点故障、客户端一致性读写等问题。这个项目用于系统化练习共识算法与后端工程组织。

实现功能

  • Raft Leader 选举与任期管理
  • 日志复制、提交与状态机应用
  • KVServer 与客户端 Clerk
  • Persister 持久化组件
  • gRPC / protobuf 通信接口
  • SkipList 底层存储结构

02 / Network Diagnosis + AI Assist

Network Tooling

AI 弱网智能检测

面向延迟、丢包、抖动、连接失败等弱网场景，把网络检测指标整理成可分析报告，并用 AI 辅助做异常判断和结果解释。

状态 本地项目，案例整理中

公开仓库整理后补充源码入口。

背景

弱网问题通常不是单一指标异常，而是延迟、丢包、抖动、链路状态共同作用。项目重点是把检测结果整理成可判断、可追踪、可复盘的工程输出。

实现功能

  • 采集并整理网络质量指标
  • 识别弱网、延迟、丢包和抖动异常
  • 输出检测分级和问题归因辅助
  • 生成检测报告，便于后续复盘
  • 为规则判断和 AI 分析预留扩展接口

技术栈

  • Python
  • Network Diagnostics
  • 异常分析
  • 数据清洗
  • 自动化脚本
  • 报告生成

Engineering Workflow

工程实现方式

先把系统边界和模块关系想清楚，再用工具提效。AI 是加速器，不替代底层理解。

01拆系统

先明确模块边界、数据流、通信方式和异常路径。

02写核心

围绕 C++、网络通信、状态管理和一致性逻辑实现主链路。

03用工具提效

Cursor / Codex 用于生成样板、辅助调试、补文档和快速迭代。

04做交付

整理 README、运行方式、截图和项目复盘，让工程能被别人验证。
